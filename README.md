# 이그니스 - 카페24 골라담기 옵션 UI

## 골라담기 문제 정의

골라담기는 같은 상품(예: 30개입)을 원하는 만큼 반복해서 담는 UX입니다.  
그런데 카페24 독립 선택형 옵션은 **같은 옵션값을 중복으로 담을 수 없습니다.** 이미 선택상품 목록에 있는 옵션을 다시 선택해도 새 상품으로 추가되지 않습니다.

이 제약을 옵션값 설계로 우회한 것입니다.

- 같은 구성을 `30개입_1`, `30개입_2`처럼 suffix만 다른 **별개 상품**으로 등록합니다. 카페24 입장에서는 서로 다른 옵션이므로 각각 담을 수 있습니다.
- 골라담기 상품 선택 커스텀 UI가 suffix를 숨기고 "30개입" 카드 하나로 묶어 보여줍니다. 카드를 누를 때마다 비어 있는 슬롯(`_1` → `_2`)을 순서대로 대신 선택해 줍니다.
- 등록된 슬롯 수가 곧 그 구성의 최대 담기 횟수가 됩니다. (`30개입_1`, `_2` → 최대 2회)

사용자에게는 "같은 상품을 두 번 골라담은" 것으로 보이고, 카페24에게는 "서로 다른 두 상품을 담은" 것이 됩니다.

## 원격 저장소 구성

카페24 에디터에서 작업하지 않고 VSCODE 코드에디터에서 SFTP로 카페24 `skin1` 스킨을 연결해서 작업했습니다.  
카페24 전체 스킨 파일을 모두 내려받지는 않았고, 이번 과제에서 실제로 수정하거나 분리한 파일만 저장소에 포함했습니다.

```txt
skin1/
  layout/basic/layout.html
  product/detail.html
  product/pick-option.html
  product/detail/*.html
  css/pick-option.css
  js/pick-option*.js
src/
  pick-option*.ts
  global.d.ts
scripts/
  deploy.mjs
```

과제와 관계없는 스킨 파일은 포함하지 않았습니다.

## 실행 / 배포

비즈니스 로직은 런타임 에러를 막기 위해 TypeScript로 작성하고, 카페24에는 컴파일된 JS만 올라가게 했습니다.

```bash
npm install
npm run build    # src/*.ts -> skin1/js/*.js
npm run deploy   # build 후 SFTP 업로드
```

`npm run deploy`는 `.vscode/sftp.json`의 접속 정보를 사용합니다. 이 파일에는 FTP 비밀번호가 들어가므로 git에는 포함하지 않았습니다.

## 배포 URL

```txt
https://guinness987.cafe24.com/product/detail.html?product_no=11
```

## 구현 방식

### 1. 골라담기 상품 선택 UI 위치

`/product/detail.html` 안의 기본 옵션 영역 앞에 아래 컴포넌트를 import했습니다.

```html
<!--@import(/product/pick-option.html)-->
```

`pick-option.html`에는 다음 내용이 들어 있습니다.

- 골라담기 UI가 들어갈 `#pickOptionArea`
- 스타일 CSS
- config / utils / cafe24 adapter / main JS 비즈니스 로직 로드

### 2. 기본 옵션은 삭제하지 않음

카페24 기본 옵션 DOM은 삭제하지 않고 숨김 처리만 했습니다.

```html
<tr class="pick-native-hidden">
  ...
</tr>
```

### 3. 옵션 그룹핑

카페24 옵션값은 아래처럼 설정되어 있습니다.

```txt
10개입_1, 10개입_2
30개입_1, 30개입_2
50개입_1, 50개입_2
100개입_1, 100개입_2
```

JS에서 suffix(`_1`, `_2`)를 뗀 이름끼리 하나로 묶습니다.

```txt
30개입_1 + 30개입_2 -> 30개입 카드, 최대 2회 담기 가능
```

추가 가능 횟수는 설정 파일에 따로 적지 않고, 실제 카페24 옵션값 개수에서 계산합니다.

### 4. 카페24와 동기화

커스텀 옵션을 클릭하면 다음 순서로 동작합니다.

1. 현재 선택상품 목록을 읽습니다.
2. 아직 사용하지 않은 다음 slot을 고릅니다. (`_1` 다음 `_2`)
3. 카페24 기본 텍스트 버튼 또는 select를 트리거합니다.
4. 카페24가 선택상품 목록/총액/장바구니 로직을 처리합니다.
5. `MutationObserver`가 선택상품 목록 변화를 감지해 커스텀 UI 상태를 다시 그립니다.

커스텀 UI가 구매 로직을 직접 구현하지 않고, 카페24 기본 흐름에 위임하는 구조입니다.

## JS 파일 역할

```txt
pick-option-config.js
  뱃지, 설명, 할인율, 문구 등 화면에 표시할 설정

pick-option-utils.js
  문자열 처리, 금액 포맷, HTML escape 같은 공통 유틸

pick-option-cafe24-adapter.js
  카페24 옵션 DOM 파싱, 선택상품 목록 읽기, 기본 옵션 트리거

pick-option.js
  UI 렌더링, 클릭 이벤트, MutationObserver 연결
```

원본 TypeScript는 `src/`에 있고, `npm run build` 결과가 `skin1/js/`에 생성됩니다.

## 상품 설정 JS

상품별 표시 데이터는 `pick-option-config.js`에서 관리합니다.

```js
window.PICK_OPTION_CONFIG = {
  cards: {
    "30개입": {
      badge: "가장 많이 사요",
      badgeTone: "orange",
      discountRate: 29,
    },
  },
};
```

설정 파일에는 화면 표시용 값만 둡니다.

## 모바일 대응

### 방식 선택: 상품 선택 영역 바로 노출

과제 요구사항은 모바일에서 "모달 방식 또는 상품 선택 영역 바로 노출 방식" 중 선택하도록 되어 있어, 바로 노출 방식을 선택했습니다.

- 상품 선택 영역과 구매 버튼 영역은 세로로 분리되어 겹치지 않습니다.

### 상품 카드 반응형

`pick-option.css`의 미디어쿼리 2단계로 처리합니다.

- 768px 이하: 카드 패딩/폰트 조정, 라디오 터치 타깃 확대
- 420px 이하: 가격 블록을 옵션명 아래 줄로 내려 좁은 폭에서도 깨지지 않게 배치

### 베이직 스킨 고정폭 해제 (모바일 폭 한정)

베이직 PC 스킨은 `body{min-width:1480px}`부터 전부 고정폭이라 모바일 표시 자체가 없고, 그대로는 상품 영역이 모바일 뷰포트 밖으로 밀려 좌우 스크롤이 생깁니다. 그래서 `pick-option.css`의 768px 미디어쿼리에서 모바일 폭 한정으로 아래를 적용했습니다.

- `layout/basic/layout.html`에 viewport 메타(`width=device-width`)를 추가했습니다. 이게 없으면 모바일 기기/에뮬레이터가 980px 가상 뷰포트로 렌더해 미디어쿼리가 아예 동작하지 않습니다.
- 스킨 고정폭 해제: `body` min-width, `#wrap`/`#container`/`#contents`, 헤더/푸터 `.inner`
- 상품 이미지/정보 2단 → 1단 스택 (`.detailArea`의 `padding-left:540px` 이미지 컬럼 자리 해제)
- 좌측 사이드바/우측 퀵메뉴는 모바일에서 숨김처리

## 캐시 관련

CSS/JS를 새로 올려도 이전 파일이 계속 보이는 경우가 있어, 배포할 때 파일명에 내용 해시를 붙였습니다.

```txt
로컬(저장소):  skin1/js/pick-option.js
서버(배포):    /js/pick-option.8c04d7.js   ← npm run deploy 가 생성
```

- `npm run deploy`가 빌드 결과물의 md5 해시(8자리)를 파일명에 넣어 업로드합니다.
- `pick-option.html` 안의 CSS/JS 경로도 배포할 때 해시 파일명으로 바꿔서 올립니다.
- 이전 배포에서 생긴 해시 파일은 배포 때 정리합니다.

## 작업 중 주의한 점

- 카페24 기본 구매/장바구니 로직은 다시 만들지 않았습니다.
- 기본 옵션 DOM은 삭제하지 않고 숨김 처리만 했습니다.
- 옵션 UI와 옵션 동작 외의 영역은 기능상 필요한 분리 외에는 수정하지 않았습니다.
- SFTP 비밀번호가 들어가는 `.vscode/sftp.json`은 git에 포함하지 않습니다.
